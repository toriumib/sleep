# Playストア公開手続きガイド — オキロ！アラーム

このリポジトリの状態に合わせた実践手順です。既にPlay Consoleアカウントをお持ちで
他アプリの公開経験がある前提で、初めての項目だけ詳しく書いています。

> **所要時間の目安**: 作業 2〜3時間 ＋ Googleの審査 数日〜2週間

---

| 項目 | 状態 |
|---|---|
| 署名済みAAB（CIで自動ビルド） | ✅ 済み |
| applicationId（`io.github.toriumib.okiroalarm`） | ✅ 済み（com.example.* はPlayが拒否するため変更済み） |
| プライバシーポリシーURL | ✅ `https://web-sigma-drab-72.vercel.app/privacy.html` |
| ストア文面（6言語） | ✅ `store/play-store-listing.md` |

| **フィーチャーグラフィック 1024×500** | ❌ 要作成（下記 §3） |

> `io.github.toriumib.okiroalarm` 以外にしたい場合は、アップロード前に

## 1. AABの入手

1. https://github.com/toriumib/sleep/actions を開く
2. 最新の成功した「Android Build (AAB)」実行を開く
3. 下部 **Artifacts** から `okiro-alarm-release-aab` をダウンロード → zipを解凍して `app-release.aab` を取り出す

以後、バージョンを上げて再提出するときは
`android/app/build.gradle.kts` の `versionCode` を +1（`versionName` も適宜）して
pushすれば、CIが新しいAABを作ります。**versionCodeを上げ忘れるとアップロードで弾かれます。**

---

## 2. Play Consoleでアプリを作成

1. https://play.google.com/console → 「アプリを作成」
2. 入力：
   - アプリ名: `オキロ！アラーム`（後で各言語のストア掲載情報で翻訳名を設定）
   - デフォルトの言語: 日本語 (ja-JP)
   - アプリ/ゲーム: **アプリ**
   - 無料/有料: **無料**（⚠️ 一度「無料」にすると有料には変更不可）
3. 申告事項にチェックして作成

---

## 3. ストアの掲載情報（審査前に全部埋める）

**「ストアでの表示」→「メインのストアの掲載情報」**

- タイトル・簡単な説明・詳しい説明: `store/play-store-listing.md` の日本語欄をコピペ
- 「翻訳を管理」→ en-US / es-419 / pt-BR / id-ID / ko-KR を追加し、同ファイルの各言語欄をコピペ

**グラフィック素材**:

| 素材 | 要件 | 用意方法 |
|---|---|---|
| アプリアイコン | 512×512 PNG（32bit、アルファ可） | `web/assets/img/icon-512.png` |
| フィーチャーグラフィック | 1024×500 PNG/JPG | `store/feature-graphic.html` をブラウザで開き、そのままスクリーンショット（下記メモ） |
| スマホスクリーンショット | 2〜8枚、縦横比 16:9〜9:16、最小辺320px | Android Studioのエミュレータでアプリを起動し、エミュレータのカメラボタンで撮影（保存先はデスクトップ） |

> フィーチャーグラフィックのメモ: `store/feature-graphic.html` はちょうど1024×500の
> デザインを表示します。Chromeで開き、F12 → Ctrl+Shift+P →「Capture node screenshot」で
> `#canvas` 要素を選ぶと正確なサイズのPNGが保存できます。

> スクリーンショットのおすすめ構成（1枚目が最重要）:
> 1枚目=鳴動画面（コインタイマー表示）、2枚目=アラーム設定、3枚目=ショップ、4枚目=名言タブ

## 4. 「アプリのコンテンツ」セクション（左メニュー → ポリシー → アプリのコンテンツ）

すべて回答しないと公開できません。本アプリの回答例：

| 項目 | 回答 |
|---|---|
| プライバシーポリシー | `https://web-sigma-drab-72.vercel.app/privacy.html` |
| アプリへのアクセス | 「すべての機能が制限なしで利用可能」（ログイン不要） |
| 広告 | **「はい、広告を含みます」** |
| コンテンツのレーティング | アンケートに回答（カテゴリ:ユーティリティ。暴力・性的内容・ギャンブル等すべて「いいえ」→ 全年齢相当になる。※コインとガチャはアプリ内通貨のみで換金不可なのでギャンブルには該当しない） |
| ターゲット層とコンテンツ | 対象年齢「13歳以上」を推奨（子ども向けにすると広告制約が厳しくなるため） |
| ニュースアプリ | いいえ |
| データセーフティ | 下記の通り |
| 政府系アプリ | いいえ |
| 金融機能 | なし |
| ヘルスケア | なし（睡眠Tipsは一般情報でヘルス機能ではない） |

**データセーフティの回答（AdMobを使うため「収集なし」にはできません）**:

1. 「データを収集または共有しますか」→ **はい**
2. 収集するデータタイプ:
   - 「デバイスまたはその他のID」→ 収集: はい ／ 共有: はい（広告目的、第三者=Google AdMob）
   - 必須/任意: 必須 ／ 暗号化して送信: はい ／ 削除リクエスト: いいえ（開発者はデータを保持しないため）
3. 目的: 「広告またはマーケティング」
4. ※アラーム設定等は端末内保存のみなので申告不要

## 5. テスト版→本番の流れ

**アカウントの注意**: 2023年11月13日以降に作成された**個人**デベロッパーアカウントは、
本番公開の前に「クローズドテストで12人以上のテスターを14日間」が必須です。
既に他アプリを本番公開済みのアカウントであれば、この要件の対象かどうかは
Play Consoleのダッシュボードに表示されます（表示がなければ不要）。

1. **内部テスト**（推奨・即時配信）:
   - 「テスト」→「内部テスト」→「新しいリリースを作成」
   - AABをアップロード → 初回に **Play アプリ署名** への登録を求められる → 「続行」でOK
     （今の鍵は「アップロード鍵」となり、Googleが配信用署名鍵を管理。鍵紛失時に再発行できるので推奨）
   - テスター（自分のGmail）を追加 → 配信リンクから実機にインストールして動作確認
   - ✅ 確認ポイント: アラームが鳴る／ロック画面に全画面表示／マナーモードで鳴る／広告表示（この時点ではテスト広告でOK）
2. **本番公開**:
   - AdMob本番IDに差し替えたAABを作る（§6）→ versionCode 2 に上げてpush → 新AABをダウンロード
   - 「製品版」→「新しいリリースを作成」→ AABアップロード
   - 「国/地域」で配信国を選択（全世界推奨。6言語のストア文面が活きます）
   - 「審査に送信」→ 審査は通常数日、新規アカウント/新規アプリは最大2週間程度

## 6. AdMob 本番ID（公開前の必須作業）

1. https://apps.admob.com → 「アプリ」→「アプリを追加」
   - プラットフォーム: Android ／「アプリはすでにアプリストアに登録されていますか」→ 公開前なので「いいえ」（公開後にPlayリスティングと紐付け）
2. 広告ユニットを2つ作成: **バナー** と **リワード**
3. 取得したIDでリポジトリの3箇所を差し替え（READMEにも記載）:
   - アプリID → `android/app/src/main/AndroidManifest.xml`
   - バナーユニットID → `android/app/src/main/res/layout/activity_main.xml`
   - リワードユニットID → `MainActivity.kt` の `REWARDED_AD_UNIT_ID`
4. `versionCode` を上げてpush → CIの新AABを本番リリースに使用
5. **公開後**: AdMobの該当アプリ設定でPlayストアのリスティングと紐付け、
   `web/app-ads.txt` の自分のpub-IDの行を有効化（ファイル内の手順コメント参照）して
   Vercelに再デプロイ、Play Consoleのストア掲載情報「ウェブサイト」に
   `https://web-sigma-drab-72.vercel.app` を設定
6. 収益の受け取りには AdMob の住所確認（PINハガキ）と支払い情報の登録が必要

> 💡 審査中〜公開直後はAdMobが「実際の広告」を配信するまで数時間〜数日かかり、
> その間は広告枠が空白になることがあります（正常です）。

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「パッケージ名が無効」 | com.example のAABを上げている → §0の新AABを使う |
| 「このversionCodeは使用済み」 | build.gradle.kts の versionCode を+1してpush |
| 「アップロード鍵が違う」 | 初回に使った鍵(`C:\Users\tori\okiro-signing\release.keystore`)で署名されたAABか確認。鍵を紛失した場合はPlay Console「設定→アプリの完全性」からアップロード鍵のリセットを申請 |
| 審査リジェクト: 全画面インテント | Android 14+ でアラーム用途は許可対象。リジェクト時は「目覚ましアプリでありUSE_FULL_SCREEN_INTENTはアラーム表示に必要」と再審査申請に記載 |
| 広告が出ない | 公開直後は配信まで時間がかかる／AdMobアカウントの承認待ちの可能性。AdMobの「アプリの準備状況」を確認 |
